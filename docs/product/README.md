# Product Telemetry & Insights

## Read First

- `telemetry-to-product.md` for product interpretation of telemetry and alerts

## Use This Folder When

- you need product meaning rather than signal semantics
- you are deciding whether an alert represents user impact
- you are prioritizing follow-up after telemetry or operations findings

## Do Not Use This Folder For

- alert thresholds or signal math
- dashboard wiring
- on-call procedures
- incident remediation
